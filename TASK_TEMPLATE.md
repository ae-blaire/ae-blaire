3. checklist 6/6 아니면 done 불가
4. 기존 상태 흐름을 깨면 안 됨

## Do

예:

- 추천 뱃지 UI 추가
- 점수순 정렬 적용

---

## Do Not
하면 안 되는 일.

예:
- 상태 전이 규칙 변경 금지
- DB 스키마 임의 변경 금지
- 기존 체크리스트 로직 제거 금지
- 서비스 키 사용 금지

---

## Done When
완료 기준을 체크리스트로 적는다.

예:
- 슬롯 후보가 점수순으로 정렬된다
- 가장 높은 후보에 추천 뱃지가 붙는다
- 추천 이유가 표시된다
- 기존 확정/삭제/추가 기능이 정상 작동한다

---

## Validation
직접 확인해야 하는 항목.

예:
- 후보 슬롯이 하나도 없을 때 UI 정상
- 선택된 슬롯이 있을 때도 추천 UI 충돌 없음
- 긴급도/중요도 값이 문자열이어도 점수 계산 가능
- 기존 상세 페이지 에러 없음
