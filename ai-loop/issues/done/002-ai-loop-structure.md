## 기대 결과
일관된 이슈 생성 / 관리 / 종료 흐름 구축

## 해결 방향
이슈 템플릿 정의 + 폴더 구조 정리 + 운영 규칙 수립

## Claude 분석

### 확인된 gap
- 이슈 파일마다 필드가 제각각임
- `reviews/` 폴더가 비어 있고 역할이 불명확함
- 이슈 파일 안에서 진행 단계 추적이 어려움
- 프롬프트 템플릿과 이슈 파일이 연결되지 않음

## 해결 방향
- 이슈 템플릿을 단일 포맷으로 통일한다
- `reviews/` 폴더는 당장 쓰지 않으면 제거하거나 용도를 명확히 한다
- 이슈 파일 내부에 단계 추적 섹션을 둔다
- 프롬프트 템플릿을 이슈 파일과 연결하는 운영 규칙을 만든다
